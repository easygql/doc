# 八、触发器

trigger主要是基于某一个对象类型的数据的增删改进行订阅，通过trigger可以非常方便的调用第三方API（比如sendgrid、slack、twillio sms等）从而实现与第三方API集成的目的。

# 8.2 触发器属性

序号 属性 含义 非空 默认值
1 id 触发器的唯一ID 非空 UUID
2 name trigger的名字 非空 无
3 typename 对应contenttype的名字 非空 无
4 eventtype Insert、 update、 remove 、all 非空 无
5 whereCondition 触发器过滤条件 可空 无
6 headers http协议头 可空 无
7 ok_status 发送成功的状态码 非空 默认为200
8 retry_times 重试次数 非空 默认为3次
9 webhook_url restapi的地址 非空 无
10 payload restapi报文体格式 可空 无

# 8.3 默认payload

假设某个contenttype的定义为如下：

type chatroom {
 content: String
 id: ID
}

对应的默认payload格式为对应的changeresult的格式：

{
eventtype: EventType
new_val: chatroom
old_val: chatroom
}

# 8.4 自定义payload

用户可以自定义trigger进行调用时payload的格式。自定义的payload为一个key-value的列表。

Key为对应返回值的对象的key值，value为返回值的JSON表述。

用户自定义trigger类型时，可以采用如下方式定义

属性 取值 说明
Key 对应key-value中的key
valuetype 枚举值，可以EnumValue，Eventvalue EnumValue为用户随便定义的某个常量EventValue为用户定义的从默认payload中取值。
value 对应的EnumValue值或者EventValue值。 当valuetype为enumValue时，value为某个固定值，当valuetype为Eventvalue时，可以取默认payload种的某个字段的值，嵌套对象用"."分隔

# 8.5  Event 对象的缓存

EasyGQL会缓存24小时以内的所有Trigger的Event数据，包括产生时间，发送时间，重试次数，并且提供手动重发的功能。EasyGQL会每24小时清空一次前24小时以前产生的Event数据。

# 8.6支持的数据库

支持rethtinkdb,postgresql,mysql。
