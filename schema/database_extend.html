# 十二、多数据库支持

本章主要讲述如何扩展新的数据库，在EasyGQL的架构下用户通过实现DaoGenerator接口来实现扩展新的数据库支持，DaoGenerator接口主要是返回一个对应数据库增、删、改、查已经订阅等接口的实现，下面以Postgresql为例，讲述如何实现扩展新的数据库。

# 12.1 DaoGenerator接口

public interface DaoGenerator {
    DataInserter getInserter();
    DataDeleter getDeleter();
    DataSelecter getSelecter();
    DataUpdater getUpdater();
    DataSub getSub();
    SchemaDao getSchemaDao();
    TriggerDao getTriggerDao();
    Many2ManyRelationCreater getMany2ManyRelationCreater();
    Many2ManyRelationRemover getMany2ManyRelationRemover();
    Many2OneRelationCreater getMany2OneRelationCreater();
    Many2OneRelationRemover getMany2OneRelationRemover();
    One2ManyRelationCreater getOne2ManyRelationCreater();
    One2ManyRelationRemover getOne2ManyRelationRemover();
    One2OneRelationCreater getOne2OneRelationCreater();
    One2OneRelationRemover getOne2OneRelationRemover();
}

每新增一类数据库，均需在"com.easygql.dao"下新增一个类使用EasyGQLDaoGenerator注解，并实现该接口，例如：

@EasyGQLDaoGenerator("PostgreSQL")
public class PostgreDaoGenerator implements DaoGenerator {
    @Override
    public DataInserter getInserter() {
       return new PostgreSqlInserter();
    }

    @Override
    public DataDeleter getDeleter() {
        return new PostgreSqLDeleter();
    }

    @Override
    public DataSelecter getSelecter() {
        return new PostgreSqlSelecter();
    }

    @Override
    public DataUpdater getUpdater() {
        return new PostgreSqlUpdater();
    }

    @Override
    public LoginDao getLogin() {
        return new PostgreSqLLogin();
    }

    @Override
    public DataSub getSub() {
        return new PostgreSqlSub();
    }

    @Override
    public SchemaDao getSchemaDao() {
        return new PostgreSqlSchema();
    }

    @Override
    public Many2ManyRelationCreater getMany2ManyRelationCreater() {
        return  new PostgreSqlMany2ManyAdd();
    }

    @Override
    public Many2ManyRelationRemover getMany2ManyRelationRemover() {
        return new PostgreSqlMany2ManyRemover();
    }

    @Override
    public Many2OneRelationCreater getMany2OneRelationCreater() {
        return new PostgreSqlMany2OneAdd();
    }

    @Override
    public Many2OneRelationRemover getMany2OneRelationRemover() {
        return new PostgreSqlMany2OneRemover();
    }

    @Override
    public One2ManyRelationCreater getOne2ManyRelationCreater() {
        return new PostgreSqlOne2ManyAdd();
    }

    @Override
    public One2ManyRelationRemover getOne2ManyRelationRemover() {
        return new PostgreSqlOne2ManyRemover();
    }

    @Override
    public One2OneRelationCreater getOne2OneRelationCreater() {
        return new PostgreSqlOne2OneAdd();
    }

    @Override
    public One2OneRelationRemover getOne2OneRelationRemover() {
        return new PostgreSqlOne2OneRemover();
    }
    @Override
    public TriggerDao getTriggerDao() {
        return new PostgreSqlTriggerDao();
    }
}

当项目启动时，系统会自动扫描com.easygql.dao包下的所有类，识别能够支持的数据库。 在以上接口中，各个方法用途如下：

  • getInserter()，返回一个DataInserter对象，每次发生数据录入操作时，EasyGQL使用该对象完成数据插入操作。
  • getDeleter()，,返回一个DataSelecter对象，每次发生数据查询操作时，EasyGQL使用该对象完成数据查询操作。
  • getDeleter()，返回一个DataDeleter对象，每次发生数据删除操作时， EasyGQL使用该对象完成具体的数据删除操作。
  • getUpdater()，返回一个DataUpdater对象，每次发生数据修改操作时，EasyGQL会使用该对象完成具体的数据修改操作。
  • getSub()，返回一个订阅流。
  • getSchemaDao()，返回一个SchemaDao，用来做Schema管理。
  • getTriggerDao()，返回TriggerDao，用来进行Trigger创建。
  • getMany2ManyRelationCreater()，返回一个Many2ManyRelationCreater用于创建多对多的关系。
  • getMany2ManyRelationRemover(),返回一个Many2ManyRelationRemover用于删除多对多关系。
  • getMany2OneRelationCreater(),返回一个Many2OneRelationCreater，用于创建多对一关系。
  • getMany2OneRelationRemover()，返回一个Many2OneRelationRemover用于删除多对一关系。
  • getOne2ManyRelationCreater()，返回一个One2ManyRelationCreater用于创建一对多的关系。
  • getOne2ManyRelationRemover()，返回一个One2ManyRelationRemover用于删除一对多的关系。
  • getOne2OneRelationCreater()，返回一个One2OneRelationCreater，用于创建一对一的关系。
  • getOne2OneRelationRemover()，返回一个One2OneRelationRemover，用于删除一对一的关系。
