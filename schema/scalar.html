# 三、标量类型

本章主要介绍现阶段EasyGQL默认支持的17种标量类型以及如何定义标量类型字段，开发者可以基于EasyGQL框架非常方便的添加新的标量数据类型。

# 3.1 17种标量类型

# 3.1.1 ID

表示这是一个唯一主键，在EasyGQL中，默认且强制每一个数据对象中，字段"id"的数据类型为“ID“，使用者不可以定义其他字段属性为“ID”，如果对象定义中没有"id"字段，EasyGQL默认会增加该字段。

# 3.1.2 Float

有符号双精度浮点值。

# 3.1.3 Boolean

布尔型数据，True或者False。

# 3.1.4 String

UTF‐8 字符序列。

# 3.1.5 Int

有符号32位整数。

# 3.1.6 Long

有符号64位整数。

# 3.1.7 Short

有符号16位整数。

# 3.1.8 Byte

字节。

# 3.1.9 BigInteger

大整形。

# 3.1.10 BigDecimal

BigDecimal。

# 3.1.11 Char

字符。

# 3.1.12 Object

任意POJO 对象。

# 3.1.13 Json

任意Json字符串。

# 3.1.14 Date

格式为yyyy-MM-dd的日期。

# 3.1.15 DateTime

ISO_OFFSET_DATE_TIME 格式的时间。

# 3.1.16 LastUpdate

yyyyMMdd HH:mm:ss格式的时间戳，EasyGQL会在每次数据更新时自动更新对象中类型为LastUpdate的字段。

# 3.1.17 CreatedAt

yyyyMMdd HH:mm:ss格式的时间戳，EasyGQL会在每次数据新增时自动更新对象中类型为CreatedAt的字段。

# 3.2 标量类型字段的定义

在EasyGQL中，一个标量类型的字段，应该具备以下属性：

序号 属性名 含义 默认值 是否非空
1 id 字段的唯一标识 系统会自动赋予每个新增字段一个唯一标识 非空
3 name 字段的名称 无 非空
3 type 字段的类型 无 非空
4 islist 是否为列表 否 非空
5 notnull 字段是否非空 否 非空
6 invisible_roles 不可查看的角色列表 空List 否
6 irrevisible_roles 不可修改的角色列表 空List 否

在EasyGQL控制台的GraphQL定义中，使用以下API进行标量的定义

type ScalarField   {
  contenttype: ContentType
  defaultvalue: String
  description: String
  id: ID
  invisible_roles: [Role]
  irrevisible_roles: [Role]
  islist: Boolean
  name: String
  notnull: Boolean
  type: ScalarType
}
type ScalarField_InsertResult   {
  id_list: String
  nodes: [ScalarField]
}
mutation {
  ScalarFieldDestroy(where: ScalarFieldWhereInput): _DeleteResult 
  ScalarFieldUpdate(object: ScalarFieldUpdateInput!, where: ScalarFieldWhereInput): _UpdateResult 
  scalarfieldsInContentType_IDInput(from_id: ID!, reset: Boolean = false, to_id: [ID]!): _NestInputResult 
  scalarfieldsInContentType_NestRemove(from_id: [ID]!, to_id: ID!): _NestRemoveResult 
  scalarfieldsInContentType_ObjectInput(from_id: ID!, reset: Boolean = false, to_object: [ScalarFieldInput]!): _NestInputResult 
}

对于标量类型，不能直接定义，必须使用嵌套对象干的添加方法，指定给某一个对象添加标量字段 以为聊天记录对象，添加具体标量字段为例：

mutation {
 	scalarfieldsInContentType_ObjectInput(from_id:"5e76ca2c158b0124a8ff7fe8",to_object:[
  	{
 		id:"5e76ca2c158b0124a8ff7ff0",   name:"roomname",
 		type:String,   description:"room name"
  },{
  id:"5e76ca2c158b0124a8ff7ff1",  name:"roominfo",
 type:String  , description:" room information"  
},{
  id:"5e76ca2c158b0124a8ff7ff2",  name:"members",
 type:Int  , description:" room information" 
 }  
]){
    inputNestIDList 
}
}

# 3.4 标量类型的扩展

开发人员可以基于EasyGQL,进行快速开发并且快速提供新的标量类型支持,具体实现可以参照LastUpdate标量类型的实现。

# 3.4.1 继承GraphQLScalarType实现标量类型


import com.easygql.component.ConfigurationProperties;
import graphql.language.StringValue;
import graphql.scalars.util.Kit;
import graphql.schema.*;

import java.time.DateTimeException;
import java.time.OffsetDateTime;
import java.time.ZonedDateTime;
import java.time.format.DateTimeParseException;
import java.util.function.Function;

public class LastUpdateScalar extends GraphQLScalarType {
    public LastUpdateScalar() {
        super("LastUpdate", "An RFC-3339 compliant timestamp updated when the object changed", new Coercing<OffsetDateTime, String>() {
            @Override
            public String serialize(Object input) throws CoercingSerializeException {
                OffsetDateTime offsetDateTime;
                if (input instanceof OffsetDateTime) {
                    offsetDateTime = (OffsetDateTime)input;
                } else if (input instanceof ZonedDateTime) {
                    offsetDateTime = ((ZonedDateTime)input).toOffsetDateTime();
                } else {
                    if (!(input instanceof String)) {
                        throw new CoercingSerializeException("Expected something we can convert to 'java.time.OffsetDateTime' but was '" + Kit.typeName(input) + "'.");
                    }

                    offsetDateTime = this.parseOffsetDateTime(input.toString(), CoercingSerializeException::new);
                }

                try {
                    return ConfigurationProperties.getInstance().DEFAULT_DATETIME_FORMAT.format(offsetDateTime);
                } catch (DateTimeException var4) {
                    throw new CoercingSerializeException("Unable to turn TemporalAccessor into OffsetDateTime because of : '" + var4.getMessage() + "'.");
                }
            }

            @Override
            public OffsetDateTime parseValue(Object input) throws CoercingParseValueException {
                OffsetDateTime offsetDateTime;
                if (input instanceof OffsetDateTime) {
                    offsetDateTime = (OffsetDateTime)input;
                } else if (input instanceof ZonedDateTime) {
                    offsetDateTime = ((ZonedDateTime)input).toOffsetDateTime();
                } else {
                    if (!(input instanceof String)) {
                        throw new CoercingParseValueException("Expected a 'String' but was '" + Kit.typeName(input) + "'.");
                    }

                    offsetDateTime = this.parseOffsetDateTime(input.toString(), CoercingParseValueException::new);
                }

                return offsetDateTime;
            }

            @Override
            public OffsetDateTime parseLiteral(Object input) throws CoercingParseLiteralException {
                if (!(input instanceof StringValue)) {
                    throw new CoercingParseLiteralException("Expected AST type 'StringValue' but was '" + Kit.typeName(input) + "'.");
                } else {
                    return this.parseOffsetDateTime(((StringValue)input).getValue(), CoercingParseLiteralException::new);
                }
            }

            private OffsetDateTime parseOffsetDateTime(String s, Function<String, RuntimeException> exceptionMaker) {
                try {
                    return OffsetDateTime.parse(s, ConfigurationProperties.getInstance().DEFAULT_DATETIME_FORMAT);
                } catch (DateTimeParseException var4) {
                    throw (RuntimeException)exceptionMaker.apply("Invalid RFC3339 value : '" + s + "'. because of : '" + var4.getMessage() + "'");
                }
            }
        });
    }
    public static final  LastUpdateScalar lastUpdateScalar=new LastUpdateScalar();
}

# 3.4.2 在EasyGQL控制台增加LastUpdate标量类型支持

在GraphQLElementGenerator.java的getScalarType方法中，增加具体的类型。

public static GraphQLScalarType getScalarType(String typeName) {
    switch (typeName) {
      case "ID":
        return Scalars.GraphQLID;
      case "Float":
      case "BigDecimal":
        return Scalars.GraphQLBigDecimal;
      case "Boolean":
        return Scalars.GraphQLBoolean;
      case "String":
      case "Char":
      case "Byte":
        return Scalars.GraphQLString;
      case "Int":
      case "Short":
        return Scalars.GraphQLInt;
      case "Long":
      case "BigInteger":
        return Scalars.GraphQLLong;
      case "Object":
        return ExtendedScalars.Object;
      case "JSON":
        return ExtendedScalars.Json;
      case "Date":
        return ExtendedScalars.Date;
      case "DateTime":
        return ExtendedScalars.DateTime;
      case "Time":
        return ExtendedScalars.Time;
      case "URL":
        return urlType;
      case "Email":
        return emailType;
      case "LastUpdate":
        return LastUpdateScalar.lastUpdateScalar;
      case "CreatedAt":
        return CreatedAtScalar.createdatscalar;
      default:
        throw new BusinessException("E10050");
    }
  }
