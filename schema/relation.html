# 七、关联关系

EasyGQL在开发者定义对象类型时，允许开发者通过定义对象类型之间的关联关系来实现定义嵌套对象类型的字段。

# 7.1 关联关系的定义

关联关系主要有以下属性：

序号 属性 含义 非空 默认值
1 id 唯一标识 是 无
2 relationtype 关联关系类型（1对1、1对多、多对1、多对多） 是 1对1
3 invisible 不可见角色列表 是 空的List
4 irrevisible 不可修改列表 是 空的List
5 description 描述 否 ""
6 fromobject 左对象类型 否 无
7 fromfield 左字段 否 无
8 toobject 右对象类型 否 无
9 tofield 右字段 否 无
10 ifcascade 是否级联删除 是 无

EasyGQL当前支持的关联关系类型有4种：1对1、1对多、多对1、多对多，具体：

  • 1对1关系，表示每一个左对象类型对象（简称“左对象”）的都可以关联到最多1个右对象类型的对象（简称“右对象”）。比如配偶关系
  • 1对多关系，表示一个左对象可以关联到多个右对象。比如父子关系
  • 多对1关系，表示多个左对象可以对应的到1个右对象。比如国籍关系
  • 多对多关系，表示每个左对象可以对应多个右对象，每个右对象又可以对应多个左对象。比如朋友关系。

在EasyGQL的关联关系种只允许对1对1和1对多的关联关系定义级联删除，当定义了级联删除后，删除左对象时会自动删除右对象。

# 7.2关联关系字段的定义

input RelationFieldInput  {
  id: ID
  fromobject:String
  fromfield:String
  toobject:String
  tofield:String
  ifcascade: Boolean = false
  invisible_roles: [Role]
  irrevisible_roles: [Role]
  relationtype: RelationType!
}
type RelationField_InsertResult  {
  id_list: String
  nodes: [RelationField]
}
type RelationFieldInput  {
  id: ID
  fromobject:String
  fromfield:String
  toobject:String
  tofield:String
  ifcascade: Boolean = false
  invisible_roles: [Role]
  irrevisible_roles: [Role]
  relationtype: RelationType!
}

relationsInSchema_IDInput(from_id: ID!, reset: Boolean = false, to_id: [ID]!): _NestInputResult 

relationsInSchema_ObjectInput(from_id: ID!, reset: Boolean = false, to_object: [RelationFieldInput]!): _NestInputResult

以上是在一个对象类型中添加一个关联关系，该对象类型在关联关系中充当左对象类型。

type RelationField_DeleteResult
  affected_rows: Int
}
relationsInSchema_NestRemove(from_id: [ID]!, to_id: ID!): _NestRemoveResult

以上是在一个对象类型中删除一个关联关系字段。

# 7.3 其他

在定义关联关系时，fromfield必须和fromobject中的其他字段，字段名不重复，tofield必须和toobject中的其他字段，字段名不重复。在关联关系字段定义完成后，会在fromobject对象类型中新增一个fromfield字段，会在toobject对象类型中新增一个tofield字段。
