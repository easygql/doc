# 九、触发器

trigger主要是基于某一个对象类型的数据的增删改进行订阅，通过trigger可以非常方便的调用第三方API（比如sendgrid、slack、twillio sms等）从而实现与第三方API集成的目的。

# 9.1 触发器的架构

如下图：

在EasyGQL中，开发者可以非常方便的针对定义的对象类型，添加触发器以集成外部Restful WebService，当Schema对应的系统用户通过EasyGQL-Engine提供的接口修改数据库数据时，EasyGQL会根据实现定义好的外部Restful API，发送服务请求。

# 9.2 触发器属性

序号 属性 含义 非空 默认值
1 id 触发器的唯一ID 非空 UUID
2 name trigger的名字 非空 无
3 typename 对应contenttype的名字 非空 无
4 eventtype Insert、 update、 remove 、all 非空 无
5 headers http协议头 可空 content-type=application/json
6 payloadformatter 发送消息内容的String格式串 非空 无
7 payloadargs 消息内容中参数值 可空 无
8 ok_status 发送成功的状态码 非空 默认为200
9 retry_times 重试次数 非空 默认为3次
10 webhook_url rest api的地址 非空 无
11 startdate 生效时间 非空 无
12 expiredate 失效时间 非空 无
13 payload restapi报文体格式 可空 无

下面以一个集成邮件发送功能的案例，来说明触发器的使用。

# 9.3 邮件发送工具

邮件发送工具代码见 github,邮件发送工具对外暴露的rest服务接口是：http://IP:端口/mail 邮件发送工具接收4个参数：

  • from: 表示发送邮箱
  • to : 表示目标邮箱
  • subject: 邮箱标题
  • text : 邮箱内容

# 9.4 定义触发器

还是以聊天室的schema为例，在定义触发器时，使用触发器定义API:

input TriggerInput   {
  description: String = ""
  eventtype: [Event]
  expiredate: Date!
  headers: Object
  id: ID
  name: String!
  ok_status: String = "200"
  payloadargs: [String]
  payloadformatter: String
  retry_times: Int = 3
  startdate: Date!
  typename: String!
  webhookurl: String!
}
mutation {
triggersInSchema_ObjectInput(from_id: ID!, reset: Boolean = false, to_object: [TriggerInput]!): _NestInputResult 
}

定义语句如下：

mutation {
  triggersInSchema_ObjectInput(from_id:"5e91e934d47ea10864002328",to_object:[
    {
      typename:"chat"
      startdate:"2019-10-11"
      expiredate:"2021-10-11"
      payloadformatter:"{\"from\":\"xxxx@gmail.com\",\"to\":\"xxxx@gmail.com\"},\"subject\":\"test email\",\"text\":\"%s\""
      eventtype:[insert]
      payloadargs:[content]
      webhookurl:"http://127.0.0.1:9996/mail"
    }
  ]) {
    inputNestIDList
  }
}

在定义好触发器后，需要重新发布Schema

mutation {
  SchemaPublish(schemaid:"5e91e934d47ea10864002328") {
    OperationResult
  }
}

如此，在聊天室插入的每条消息,对应邮箱都会收到一封通知邮件
