# 九、权限控制

EasyGQL可以为开发者提供根据用户不同进行不同行级、列级权限控制的API。

# 9.1权限控制流程

如上图所示，EasyGQL的权限控制是通过权限控制过滤器来事先的，当用户请求数据时，权限过滤器会根据用户当前的基本信息和开发者事先定义好的数据方位控制规则，判断用户是否对该API具有访问权限，以及是否对应的行、列具有访问权限，从而决定是否调用数据库。

  1. 当用户不具备访问API权限时，用户请求将会被返回一个403错误。
  2. 当用户具备访问API权限时，用户只能查看或修改经过权限过滤器过滤后的数据。

# 9.2 权限控制模型

如上图所示，Easy的权限控制模型主要是如下关系。

  1. 每一个数据类型和用户角色可以定义不被允许访问的API。

  2. 每一个数据类型和用户角色可以针对针对允许访问的API设置数据过滤条件。数据过滤条件氛围：列级和行级权限。

  3. 默认情况下，每个用户角色都有所有API的访问权限，以及所有行和所有列的访问权限。

  4. 列级数据过滤条件主要是设置用户不允许访问的数据权限。

  5. 行级数据过滤条件是根据用户的基本信息设置数据行的过滤条件。

# 9.3 针对不同的API设置不同的数据访问权限

用户可以针对EasyGQL提供的各个API设置不同的数据访问权限。在开发者新建或修改数据类型定义的时候，默认为为每个数据类型提供以下几种API。

序号 API名称 用法 权限控制种类
1 类型名 根据ID查询具体某个对象 行级+列级权限控制，只能查询指定行、指定列
2 类型名+”S” 根据where条件查询多个对象 行级+列级权限控制，只能查询指定行、指定列
3 类型名+”Create” 创建对象 列级权限控制，在创建对象时，只能对某些固定的列赋值。
4 类型名+”Destroy” 销毁对象 行级权限控制，只能删除指定的行
5 类型名+”Update” 更新对象 行级+列级权限控制，只能针对指定行、指定列进行修改。
6 类型名+”Listener” 订阅对象变化 行级+列级权限控制，只能针对指定的行、指定的列订阅变化。
7 字段名+”In”+类型名+”Add” 为对象创建内嵌对象 行级+列级权限控制，参照update的权限。只能修改指定行，指定列。
8 字段名+”In”+类型名+”Remove” 销毁对象的内勤对象。 行级+列级权限控制，参照update的权限。只能修改指定行，指定列。

# 9.4 列级权限控制

EasyGQL默认为所遇API提供所有列的查询和操作权限，在进行列级权限控制时，可以针对每个类型、每个角色、每个API具体禁止操作列的列表。

# 9.5 行级权限控制

行级权限控制是EasyGQL可以根据用户对属性不同，为不同用户提供不同的数据库操作结果集。

针对每一个自定义数据类型，系统会自动提供其查询过滤器（数据类型为类型名+”WhereInput”），开发者可以通过设定默认的过滤器参数，实现行级过滤。

默认过滤器设置的输入规则总体需要符合对应对象的WhereInput的定义，但是对于叶子节点的参数，不再是EasyGQL默认提供的标量类型，而是以下类型

字段名 取值 含义
value 对应过滤器叶子节点的值 当valuetype为const时，value为对应叶子节点的常量值得json表述。当valuetype为userdata时，value为user的某一个字段名。
valuetype “const”、”userdata”

假设某一个对象定义如下：

type  chatroom {
   id :ID
   content :String
}

对应对象的WhereInput描述为

input UserWhereInput {
 	and: UserWhereInput 
field_contains: [String] 
filter: UserFieldFilter 
not: UserWhereInput 
or: UserWhereInput 
}
input chatroomFieldFilter { 
content: StringFilter 
id: IDFilter
}

这对应的行级过滤条件可以为：

{ 
  “or”:[
  	{
 	“filter”:{
      “content”:{
          “eq”:{
             “valuetype”:”const”,
             “value”:”test11”
}
}
}
},
		{
 		“filter”:{
      	“content”:{
        	  “eq”:{
            	 “valuetype”:”userdate”,
             	 “value”:”role”
				   }
				}
			}
}
]
}

表示，可以对content是test11或者content值等于用户的role的所有对象。
