# 十、外部Schema的集成

为了增加EasyGQL的扩展性，EasyGQL提供集成外部Schema的功能。

# 10.1 外部Schema的定义

序号 属性 含义 非空 默认值
1 id 唯一标识 是 无
2 endpoint 外部Schema的端点 是 无
3 apilist 应用的API列表 是 空的List
4 headers 对应http协议报文头 是 空的map
type RemoteSchema {
   id:ID
   endpoint:String
   apilist:[RemoteAPI]
   headers:object
}
type RemoteAPI {
   id:ID
   apiname:String
   disabled_roles:[Role]
}

# 10.2 外部Schema集成方法

  • 每一个外部schema都可以用一个URL表示的endpoint来代替，EasyGQL能够解析并识别对应Schema的定义，开发者可以引用对应外部Schema的api，来实现多个Schema集成的目的。
  • schema会为每一个外部schema的endpoint生成一个16位的唯一字符串(简称endpointid)，当应用了该schema的api时，会对应的在本schema中生成一个apiname+""+endpointid的api，并且与该api相关的所有数据类型也都会被引入到当前Schema,并将原来schema中的对应对象类型名字改为类型名+""+endpointid。
  • 每当被引用的api被调用时，EasyGQL会调用对应的外部Schema，并将它的返回值返回给调用方。

# 10.3 外部Schema集成的API


input RemoteSchemaInput {
   id:ID
   endpoint:String!
   headers:object
}
input RemoteAPIInput {
  id:ID
  apiname:String!
  disabled_roles:[Role]
}
remoteschemaInSchemaObjectAddInput(child_id:[ID],child_object:[RemoteSchemaInput],confilctEnum:ConfilctEnum=Replace,parent_Id:ID!):RemoteSchema_InsertResult
remoteschemaInSchemaObjectRemove(child_id:[ID],parent_Id:ID!):RemoteSchema_DeleteResult
apilistInRemoteSchemaAddInput(child_id:[ID],child_object:[RemoteAPIInput],parent_Id:ID!):RemoteAPI_InsertResult
apilistInRemoteSchemaRemove(child_id:[ID],parent_Id:ID!):RemoteAPI_DeleteResult
