# 八、触发器（Trigger)

trigger主要是基于某一个contenttype的某一个Subscription的事件进行订阅，通过trigger可以非常方便的调用第三方API（比如sendgrid、slack、twillio sms等）从而实现与第三方API集成的目的。

# 8.2 触发器属性

属性 取值 备注
Trigger name trigger的名字 一个schema中trigger的名字必须唯一
contenttype 对应contenttype的名字 对应于哪个contenttype的订阅。
Eventtype Insert、 update、 remove 、all 订阅的类型
Where 订阅的过滤条件 对应subscription的where 条件
changefields 对应对象字段的列表 只有当对应列变了才会触发。
Headers 一个map对象 表示用于调用第三方API时说附带的http请求头
Ok_status 2XX开头的某个状态码 表示为请求调用成功，默认为200
payload 定义trigger调用时的payload 默认为订阅时的返回值。
Retry_times 整形，重试次数，不得大于10次

# 8.3 默认payload

假设某个contenttype的定义为如下：

type chatroom {
 content: String id: ID 
}

对应的默认payload格式为对应的changeresult的格式：

type chatroomChangeResult { 
eventtype: EventType 
new_val: chatroom 
old_val: chatroom 
}

# 8.4 自定义payload

用户可以自定义trigger进行调用时payload的格式。自定义的payload为一个key-value的列表。

Key为对应返回值的对象的key值，value为返回值的JSON表述。

用户自定义trigger类型时，可以采用如下方式定义

属性 取值 说明
Key 对应key-value中的key
valuetype 枚举值，可以EnumValue，Eventvalue EnumValue为用户随便定义的某个常量EventValue为用户定义的从默认payload中取值。
value 对应的EnumValue值或者EventValue值。 比如，“1”为EnumValue值。Eventtype,以及new_val.content表示new_val下的content字段的取值。

# 8.5 Event 对象的缓存

EasyGQL会缓存24小时以内的所有Trigger的Event数据，包括产生时间，发送时间，重试次数，并且提供手动重发的功能。EasyGQL会每24小时清空一次前24小时以前产生的Event数据。

# 8.6支持的数据库

支持rethtinkdb,postgresql,mysql。
