# 四、自定义类型

# 4.1 自定义类型的概念

自定义类型的概念是用以区别标量类型的，标量类型是原子类型，自定义类型是用户勾选的标量类型以及自定义类型的集合。

# 4.2 自定义类型的字段

每个自定义类型都默认会自带一个id字段，类型为ID，其余字段为用户自己选择，CreatedAt和lastupdate字段，同一个自定义对象不能有两个。

# 4.3.1 对象的新增

针对用户定义的每一个对象，EasyGQL会自动提供一个名为“对象名Create”的接口，输入参数为对象的数组和conflict配置，如下所示：

chatCreate(conflict: String = "replace", objects: [chatInput]): insert_return
Createchatroom(conflict: String = "replace", objects: [chatroomInput]): insert_return

conflict是一个冲突解决配置，默认为replace，表示当碰到ID重复的数据对象时进行整体替代，其他可选择值为update，update表示对冲突对象中重复的字段进行更新，原有的字段如果在插入的时候没有被声明将会保留。 objects 是对应对象的一个数组，对应输入对象的类型为“对象名Input”,例如：

input chatInput {
  chatroom: ID!
  from: String!
  id: ID
  message: String!
}
input chatroomInput {
  id: ID
  owner: String!
}

进行数据插入时，对应输入类型与之前定义的类型区别主要有：

  • 增加了一个可选字段id,如果插入时不给该字段赋值，EasyGQL会自动赋予一个UUID为该数据对象的唯一主键。
  • 在存在对象嵌套的情况下，用对应对象的id表示该对象。

进行数据插入时返回值的格式如下有两部分组成：affectedrow表示被影响的行数,idlist表示插入数据的id列表：

type insert_return {
  affectedrow: Int
  idlist:[ID]
}

此外，EasyGQL会了避免前端长时间等待后台操作结果的返回，提供了不带任何返回值的新增接口，结构如下:

chatCreateNoReply(conflict: String = "replace", objects: [chatInput]): Boolean
chatroomCreateNoReply(conflict: String = "replace", objects: [chatroomInput]): Boolean

# 4.3.2 对象的删除

针对用户定义的每一个对象，EasyGQL会提供一个名为"对象名Destory"数据删除接口，输入参数是查询条件。例如：

chatDestory(where: chatWhereFilter): delete_return
chatroomDestory(where: chatroomWhereFilter): delete_return

参数where为数据查询时的过滤条件，满足过滤条件的所有对象均会被删除。 数据返回有三部分数据：affectedrow表示被影响的行数。例如：

type delete_return {
  affectedrow: Int
}

此外，EasyGQL会了避免前端长时间等待后台操作结果的返回，提供了不带任何返回值的删除接口，结构如下:

chatDestoryNoReply(where: chatWhereFilter): Boolean
chatroomDestoryNoReply(where: chatroomWhereFilter): Boolean

# 4.3.3 对象的修改

针对用户定义的每一个对象，EasyGQL会提供一个"Update对象名"的数据更新接口，该接口接受两个参数：where表示查询条件，object表示待更新的值。例如：

chatUpdate(object: chatUpdateInput!, where: chatWhereFilter): update_return
chatroomUpdate(object: chatroomUpdateInput!, where: chatroomWhereFilter): update_return

object的对象类型是“对象名UpdateInput”，是必填字段。“对象名UpdateInput”与"对象名input"的区别是所有字段均为非必填字段。例如：

input chatUpdateInput {
  chatroom: ID
  from: String
  id: ID
  message: String
}
input chatroomUpdateInput {
  id: ID
  owner: String
}

返回值有三部分内容：affectedrow表示更新的行数，例如：

type update_return {
  affectedrow: Int
}

此外，EasyGQL会了避免前端长时间等待后台操作结果的返回，提供了不带任何返回值的修改接口，结构如下:

chatUpdateNoReply(object: chatUpdateInput, where: chatWhereFilter): Boolean
chatroomUpdateNoReply(object: chatroomUpdateInput, where: chatroomWhereFilter): Boolean

# 4.3.4 对象的查找

针对用户定义的每一个对象类型，EasyGQL提供两个查询接口：“对象名s”和“对象名”。

# 4.3.4.1 "对象名s"

如下是chats和chatrooms的定义:

chatrooms(first: Int = -1, last: Int = -1, limit: Int = 2147483647, orderby: String, skip: Int = 0, where: chatroomWhereFilter): chatroomwhere_result chats(first: Int = -1, last: Int = -1, limit: Int = 2147483647, orderby: String, skip: Int = 0, where: chatWhereFilter): chatwhere_result

在以上定义中：

  • first表示按照当前排序规则取前xx个要素。
  • last表示按照当前排序规则取最后xx个要素。
  • limit表示按照当前排序规则取最多XX个要素。limit不能和first、last共用。
  • skip表示按照当前排序规则从第XX个元素开始取值。skip不能和first、last共用，skip可以和limit进行组合分页。
  • orderby表示查询时的排序规则，使用方法如“chatroom,desc”，表示对chatroom进行倒序排列。
  • "对象名WhereInput“表示对象的过滤条件。

过滤条件对象“对象名WhereInput”：

input chatWhereInput {
  and: chatWhereFilter
  filter: chatFieldFilter
  field_contains: [String]
  not: chatWhereFilter
  or: chatWhereFilter
}
input chatroomWhereInput {
  and: chatroomWhereFilter
  filter: chatroomFieldFilter
  field_contains: [String]
  not: chatroomWhereFilter
  or: chatroomWhereFilter
}
input chatFieldFilter {
  chatroom: IDFilter
  from: StringFilter
  id: IDFilter
  message: StringFilter
}
input chatroomFieldFilter {
  id: IDFilter
  owner: StringFilter
}

其中：

  • and,or,not表示对后续过滤条件的与、或、非操作。
  • field_contains，表示所查询对象必须包含的字段。
  • "对象名FieldFilter"是所查询对象的字段级过滤条件，根据每个字段所基于的标量类型不一样，查询过滤条件不一样。具体见下表：
标量类型 操作符
String eq,ne,lt,le,gt,ge,match
Int eq,ne,lt,le,gt,ge
Long eq,ne,lt,le,gt,ge
BigDecimal eq,ne,lt,le,gt,ge
ID eq,ne,lt,le,gt,ge,match
Boolean istrue
List hasone,none,all

返回结果定义如下：

type chatroomwhere_result {
  result: [chatroomOutput]
}
type chatwhere_result {
  result: [chatOutput]
}
type chatroomOutput {
  createdat: DateTime @createdat
  id: ID
  owner: String
}
type chatOutput {
  chatroom: chatroomOutput
  createdat: DateTime @createdat
  from: String
  id: ID
  message: String
}

# 4.3.4.2 "对象名"

操作定义如下：

chat(id: ID!): chatOutput
chatroom(id: ID!): chatroomOutput

这是根据ID，返回一个确定对象的方法。

# 4.3.5 对象的订阅

自定义对象的订阅接口是自定义对象的实时API，能够根据确定的查询条件，返回指定对象的变更。

chatListener(event: [EventType], where: chatWhereFilter,on:ValueType): chatSubscriptionResult
chatroomListener(mutationtypes: [String], where: chatroomWhereFilter): chatroomSubscriptionResult

订阅接口的输入参数中：

  • event是操作类型的列表，有insert、update、delete三种。
  • where是订阅对象的查询条件。
  • ValuetType表示监听old_val还是监听new_val。

返回结果定义如下：

type chatChangeResult {
  _typename: String
  evnenttype:EventType
  old_val:Chat
  new_val:Chat
}
type chatroomSubscriptionResult {
  _typename: String
  evnenttype:EventType
  old_val:Chatroome
  new_val:Chatroom
}
