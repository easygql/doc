# 九、自定义API

# 9.1自定义API的扩展

EasyGQL是一个开源的GraphQL框架，它允许开发者在它基础上新增一些自己开发的API，但它必须要满足以下约束：

  • 自定义API必须要使用EasyGQLThirdAPIAnnotation注解，并注明API的名字，比如:
@EasyGQLThirdAPIAnnotation("UserLogin")
  • 自定义API必须要继承ThirdAPI这个基类，且必须要重载dowork、inputFieldS、outputFields三个方法。其中dowork是实际执行API时调用的方法，inputFields和outputFields分别定义输入和输出参数。
  • 自定义API在执行时，会传入一个ThirdAPIInput类型的对象，该对象主要包含schema的ID(schemaID)，shcema的基本信息（schemaData）、用户的基本信息（userInfo）、用户定义的输入参数的值(runTimeInfo)
public class ThirdAPIInput {
    private String schemaID;
    private SchemaData schemaData;
    private HashMap userInfo;
    private HashMap runTimeInfo;
}

下面时一个自定义API的示例:

package com.easygql.thirdapis;

import com.easygql.util.GraphQLCache;
import com.easygql.util.ScalarFieldInfo;
import com.easygql.util.ThirdAPIInput;

import java.util.HashMap;

import static com.easygql.config.ConfigurationProperties.*;


@EasyGQLThirdAPIAnnotation("UserLogin")
public class UserLogin extends  ThirdAPI {
    @Override
    public Object doWork(ThirdAPIInput thirdAPIInput) {
        String username = (String)thirdAPIInput.getRunTimeInfo().get(GRAPHQL_USERNAME_FIELD);
        String password = (String)thirdAPIInput.getRunTimeInfo().get(GRAPHQL_PASSWORD_FIELD);
        return  GraphQLCache.getEasyGQL(thirdAPIInput.getSchemaID()).getLoginDao().login(username,password);
    }

    @Override
    public HashMap<String, ScalarFieldInfo> inputFields() {
        ScalarFieldInfo userNameField = new ScalarFieldInfo();
        userNameField.setName(GRAPHQL_USERNAME_FIELD);
        userNameField.setDescription("User Name");
        userNameField.setType(GRAPHQL_STRING_TYPENAME);
        userNameField.setNotnull(true);
        ScalarFieldInfo passwordField = new ScalarFieldInfo();
        passwordField.setName(GRAPHQL_PASSWORD_FIELD);
        passwordField.setNotnull(true);
        passwordField.setType(GRAPHQL_STRING_TYPENAME);
        passwordField.setDescription("PassWord");
        HashMap<String,ScalarFieldInfo> inputFieldsMap = new HashMap<>();
        inputFieldsMap.put(GRAPHQL_USERNAME_FIELD,userNameField);
        inputFieldsMap.put(GRAPHQL_PASSWORD_FIELD,passwordField);
        return inputFieldsMap;
    }

    @Override
    public HashMap<String, ScalarFieldInfo> outputFields() {
        ScalarFieldInfo tokenField = new ScalarFieldInfo();
        tokenField.setName(GRAPHQL_TOKEN_FIELDNAME);
        tokenField.setDescription("Token");
        tokenField.setType(GRAPHQL_STRING_TYPENAME);
        tokenField.setNotnull(true);
        HashMap<String,ScalarFieldInfo> outputFieldsMap = new HashMap<>();
        outputFieldsMap.put(GRAPHQL_TOKEN_FIELDNAME,tokenField);
        return outputFieldsMap;
    }

    @Override
    public String getAPIType() {
        return GRAPHQL_MUTATIONAPI_FIELDNAME;
    }

}

EasyGQL框架会在程序启动时检查com.easygql.thirdapis这个包下面的所有类以及Jar包，将所有符合条件的自定义API识别出来。

# 9.2 自定义API的使用

在创建Schema时，可以指定集成哪些自定义API

input ThirdAPIInput {
  apiname: String!
  disabled_roles: [[Role]]
  id: ID
}
thirdapisInSchemaObjectAddInput(child_Id: [ID], child_object: [ThirdAPIInput], conflict: CONFLICTEnum = replace, parent_Id: ID!): ThirdAPI_InsertResult
thirdapisInSchemaObjectRemove(child_Id: [ID], parent_Id: ID!): ThirdAPI_DeleteResult

Schema中添加了相应的自定义API后，会为schema生成相应apiname和参数列表的api，并且提供相应的权限控制功能。
