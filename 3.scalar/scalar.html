# 三、标量类型

本章主要介绍现阶段EasyGQL默认支持的17种标量类型以及如何定义标量类型字段，开发者可以基于EasyGQL框架非常方便的添加新的标量数据类型。

# 3.1 17种标量类型

# 3.1.1 ID

表示这是一个唯一主键，在EasyGQL中，默认且强制每一个数据对象中，字段"id"的数据类型为“ID“，使用者不可以定义其他字段属性为“ID”，如果对象定义中没有"id"字段，EasyGQL默认会增加该字段。

# 3.1.2 Float

有符号双精度浮点值。

# 3.1.3 Boolean

布尔型数据，True或者False。

# 3.1.4 String

UTF‐8 字符序列。

# 3.1.5 Int

有符号32位整数。

# 3.1.6 Long

有符号64位整数。

# 3.1.7 Short

有符号16位整数。

# 3.1.8 Byte

字节。

# 3.1.9 BigInteger

大整形。

# 3.1.10 BigDecimal

BigDecimal。

# 3.1.11 Char

字符。

# 3.1.12 Object

任意Java对象。

# 3.1.13 Json

任意Json字符串。

# 3.1.14 Date

格式为yyy-MM-dd的日期。

# 3.15 DateTime

ISO_OFFSET_DATE_TIME 格式的时间。

# 3.16 LastUpdate

yyyyMMdd HH:mm:ss格式的时间戳，EasyGQL会在每次数据更新时自动更新对象中类型为LastUpdate的字段。

# 3.17 CreatedAt

yyyyMMdd HH:mm:ss格式的时间戳，EasyGQL会在每次数据新增时自动更新对象中类型为CreatedAt的字段。

# 3.2 标量类型字段对的定义

在EasyGQL中，一个标量类型的字段，应该具备以下属性：

序号 属性名 含义 默认值 是否非空
1 id 字段的唯一标识 系统会自动赋予每个新增字段一个唯一标识 非空
3 name 字段的名称 无 非空
3 type 字段的类型 无 非空
4 islist 是否为列表 否 非空
5 notnull 字段是否非空 否 非空
6 invisible_role 不可查看的角色列表 空List 否
6 unmodifiable_role 不可查看的角色列表 空List 否
